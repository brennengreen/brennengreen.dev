edgecortex: an intelligence proxy that makes any small local model reliably agentic

A transparent, OpenAI-compatible reverse proxy that sits between any client and any local model server — and quietly fixes the model instead of the client.

Published May 6, 2026 By Brennen Green Read ~11 min
RustLocal LLMOpenAI APIMiddlewareGBNF

Every local model has quirks. Bonsai 8B drops opening braces. Qwen3 0.5B emits bare JSON with no XML tags. SmolLM2 repeats itself into a loop. Gemma 4 stops "thinking" too early. And every agentic harness in the world rebuilds the same brittle workarounds for these, badly, from scratch. edgecortex is the other approach: a smarter pipe that repairs the model's output transparently, so every client that points at it gets a more reliable model for free.

Fix the model, not the client

The usual way to make a small model behave is to bury the workarounds inside your agent: regex to rescue malformed JSON, a retry loop here, a repetition guard there, a per-model sampling table somewhere else. It works until you switch models or build a second client, and then you do it all again. The insight behind edgecortex is that all of these fixes belong in one place — between the client and the backend — where they're written once and benefit everything.

It speaks the OpenAI chat-completions API on both sides. Point any OpenAI-compatible client (Continue, a custom agent, plain curl) at edgecortex instead of the backend, point edgecortex at any local server (ollama, llama.cpp, vLLM), and small-model reliability improves with zero changes on either end.

The intelligence pipeline

A request doesn't just get forwarded — it passes through a pipeline that shapes it on the way in and repairs it on the way out. Each stage activates conditionally, based on the request and the detected model, so a well-behaved strong model pays almost nothing while a quirky tiny one gets the full treatment.

Model profiles: a quirk database

The heart of edgecortex is a registry of per-model profiles. Each profile encodes a model's optimal sampling parameters and — critically — the specific failure modes it's known to exhibit, so the right repairs activate automatically the moment a known model name shows up in a request.

ModelSizeQuirks handled
Gemma 4 E2B / E4B1.5–3.0 GB—
Qwen3 0.5B0.5 GBbare_json, premature_think, repetition
SmolLM2 1.7B1.0 GBbare_json, premature_think, repetition
Bonsai 8B1.2 GBmissing_brace, token_doubling, premature_think, repetition
Qwen3 Coder 8B5.0 GB— (strong)

A profile is the difference between "this model is broken" and "this model needs its braces fixed and its sampling pinned." When Bonsai 8B is detected, edgecortex knows to expect dropped opening braces and doubled tokens — and turns on exactly the repairs that address them.

Making structure non-negotiable

Agentic work lives or dies on structured output. edgecortex enforces it with three layers of defense, in increasing order of cost:

  • Grammar enforcement. For tool-calling and structured requests, edgecortex hands the backend a GBNF grammar that constrains generation to valid shapes — the model literally cannot emit a token that breaks the structure.
  • JSON normalization. For models or backends where grammar isn't available, a normalizer repairs the common pathologies after the fact: a missing opening brace, bare JSON the model forgot to wrap, trailing commentary.
  • Structured retry. If a response that's supposed to be JSON still fails to parse, edgecortex validates and automatically retries up to twice before giving up — turning a transient malformation into a non-event for the client.

On the streaming path, two more guards run continuously: a token de-duplicator removes the doubled tokens that models with the token_doubling quirk emit, and a degenerate-output detector aborts a response the moment it detects runaway repetition — so a model that falls into a loop doesn't burn your battery generating the same sentence 400 times.

Adapting to the hardware

The same proxy runs on a Raspberry-Pi-class box and a workstation, so it auto-detects the machine at startup and picks a device tier that sets how aggressively it scaffolds and how large a token budget it spends:

TierRAMBehavior
1≤ 2 GBMaximum scaffolding, 512-token budget, aggressive compaction
22–4 GBHeavy scaffolding, 2048-token budget
34–8 GBModerate scaffolding, 4096-token budget
48 GB+Light touch, 8192-token budget

Conversation compaction is the lever that makes the small tiers viable: when a history grows past the model-aware threshold, edgecortex compresses older turns into summaries so the working context stays inside the budget — the same "don't let the prompt grow unbounded" principle that aircoder takes to its logical extreme.

An optional richer API

Standard clients just see a better OpenAI endpoint. Smarter clients can opt into the /v1/cortex/* surface, which exposes the proxy's reasoning as first-class endpoints — classify a task's complexity, get an execution plan, or read the full pipeline status:

# Point any OpenAI client at the proxy — quirks are repaired transparently
curl http://localhost:8080/v1/chat/completions \
  -d '{"model":"gemma-4-e2b","messages":[{"role":"user","content":"hello"}]}'

# Or use the enhanced Cortex endpoints for richer interaction
curl -X POST http://localhost:8080/v1/cortex/classify \
  -d '{"messages":[{"role":"user","content":"Build a REST API"}],"model":"gemma-4-e2b"}'

curl http://localhost:8080/v1/cortex/status   # full intelligence-pipeline state

Under the hood it's a six-crate Rust workspace — a protocol crate of shared types, an OpenAI HTTP/SSE client, a tools layer, an agent runtime, the original CLI, and the proxy itself, where the profiles, classifier, compaction, retry, and stream-filter modules live.

Why a proxy is the right shape

edgecortex is "picks and shovels" for the local-AI gold rush. Instead of building yet another CLI that competes with Claude Code or Copilot, it builds the infrastructure layer that makes every local model better — decoupled from any one model, usable by any client, written once.

The mission is concrete: make it possible for anyone, on any hardware, to use a fully local LLM for serious agentic coding — and remove the dependence on massive cloud providers to do it. A frontier model doesn't need an intelligence layer wrapped around it. A 1.2 GB model that drops braces and loops on itself absolutely does, and that's exactly the model most people can actually run on the hardware they own. edgecortex meets the model where it is and hands the client something it can trust.

edgecortex is an MIT-licensed, six-crate Rust workspace. It's the middleware companion to aircoder (the agent) and metalstream (the runtime) — three angles on the same goal: frontier-quality agentic coding, fully local, on the machine you already own.
